1. Мета роботи

Набути практичних навичок зі створення динамічних веб-сторінок з використанням JavaScript для відображення та оновлення даних моніторингу енергетичних об'єктів у режимі реального часу.

Завдання:

  1. Спроєктувати структуру веб-сторінки для моніторингу енергетичного об'єкта.
  2. Реалізувати динамічне оновлення параметрів за допомогою JavaScript.
  3. Додати можливість ручного оновлення (кнопки).
  4. Реалізувати автоматичне оновлення (таймер).
  5. Додати візуальну індикацію стану параметрів.
  6. Протестувати роботу сторінки.

2. Теоретична частина

2.1. DOM та робота з елементами

DOM (Document Object Model) — програмний інтерфейс для HTML-документів, що представляє сторінку як дерево об'єктів. Кожен HTML-елемент є вузлом дерева, до якого можна отримати доступ та змінити його програмно.

Основні методи доступу до елементів:

  • document.getElementById('id') — пошук за ідентифікатором;
  • document.querySelector('.class') — пошук за CSS-селектором;
  • document.querySelectorAll('.class') — пошук усіх відповідних елементів;
  • document.createElement('tag') — створення нового елемента.

Зміна вмісту:

  • element.textContent — зміна текстового вмісту (безпечно);
  • element.innerHTML — зміна HTML-вмісту (може мати XSS-ризики);
  • element.classList.add/remove/toggle — керування CSS-класами;
  • element.style.property — зміна inline-стилів.

2.2. Події та таймери

Події (Events) дозволяють реагувати на дії користувача. Для додавання обробників використовується метод addEventListener(type, handler). Типові події: click, change, DOMContentLoaded.

JavaScript надає два методи для роботи з таймерами:

  • setTimeout(fn, ms) — виконати функцію один раз через затримку;
  • setInterval(fn, ms) — виконувати функцію періодично;
  • clearInterval(id) — зупинити періодичне виконання.

У даній роботі setInterval використовується для автоматичного оновлення даних моніторингу з можливістю зупинки через clearInterval.

2.3. Генерація випадкових чисел

Для імітації даних датчиків використовується Math.random() — генерує число від 0 до 1. Для отримання значення у заданому діапазоні [min, max]: 

function getRandom(min, max, decimals) {
    const value = Math.random() * (max - min) + min;
    return parseFloat(value.toFixed(decimals));
}

2.4. Опис об'єкта автоматизації

Варіант 15: Насосна станція водопостачання. Об'єкт призначений для подачі питної та технічної води до споживачів міського району. Обладнана 4 відцентровими насосами з частотно-регульованим приводом.

Перелік вимог до веб-сторінки:

  • Ручне та автоматичне оновлення;
  • Графіки зміни параметрів у часі;

3.2. Проєктування

Структура HTML

Сторінка складається з таких семантичних блоків:

  • <nav> — навігація з Bootstrap navbar;
  • <header> — герой-секція з назвою, годинником, статусом системи;
  • <section class="controls-bar"> — панель керування (кнопки, інтервал);
  • <section id="dashboard"> — картки 4-х основних параметрів (генеруються JS);
  • <section id="extra"> — додаткові параметри (насоси, обертання, температура);
  • <section id="charts"> — 4 графіки Chart.js;
  • Секція журналу вимірювань із таблицею;
  • <section id="about"> — опис об'єкта;
  • <footer> — нижній колонтитул.

План стилізації

  • Кольорова схема: водна тематика (блакитний #0077b6, синій #023e8a, аква #00b4d8);
  • CSS-змінні для кольорів та розмірів;
  • Bootstrap 5 grid для адаптивності;
  • Градієнти для navbar, hero, footer;
  • Анімації: fadeInUp для секцій, pulse для критичних статусів;
  • Медіа-запити для мобільних (576px, 768px, 992px).

Алгоритм роботи JavaScript

  1. При завантаженні сторінки (DOMContentLoaded):
    • побудова карток параметрів (buildParamCards);
    • ініціалізація 4-х графіків Chart.js (initCharts);
    • завантаження журналу з localStorage;
    • перше оновлення даних (updateAll);
    • запуск годинника (setInterval(updateClock, 1000));
    • додавання обробників подій на кнопки.
  2. Функція updateAll():
    • генерація випадкових даних для кожного параметра;
    • оновлення DOM-елементів (значення, клас статусу, прогрес-бар);
    • оновлення додаткових параметрів (насоси, обертання, температура);
    • визначення загального статусу системи;
    • оновлення графіків Chart.js;
    • запис у журнал + localStorage.
  3. Автооновлення: setInterval(updateAll, intervalMs) з можливістю зупинки.

4. Тестування

4.1. Перевірка генерації даних

Перевірено, що всі згенеровані значення знаходяться в межах встановлених діапазонів. Тиск завжди в межах 3–6 атм, продуктивність 50–200 м³/год, потужність 0–100 кВт, рівень води 0–10 м. Точність відповідає параметру decimals.

4.2. Перевірка оновлення відображення

При натисканні кнопки «Оновити» всі 4 картки параметрів оновлюються новими значеннями. Відповідно оновлюються числові значення, прогрес-бари, кольорові індикатори статусу. Додаткові параметри (насоси, обертання, температура) також оновлюються.

4.3. Перевірка індикації статусів

СтанКолірУмова
Норма ● Значення в межах normal[0] – normal[1]
Попередження ● Значення в межах min – max, але за межами норми
Критично ● Значення за межами діапазону min – max

4.4. Перевірка таймера

Автооновлення коректно вмикається та вимикається. Зміна інтервалу (2, 3, 5, 10 сек) працює в реальному часі — при зміні значення <select> таймер перезапускається з новим інтервалом. Статус-бейдж відображає поточний стан.

4.5. Перевірка додаткових функцій

  • Графіки: Chart.js коректно відображає історію параметрів, обмежуючи кількість точок до 20;
  • Журнал: записи додаються до таблиці, зберігаються в localStorage, відновлюються при перезавантаженні;
  • Експорт CSV: файл завантажується з коректною кирилицею (BOM) та роздільником «;»;
  • Адаптивність: сторінка коректно відображається на мобільних пристроях (перевірено Chrome DevTools).

5. Висновки

Що було реалізовано:

  • Інтерактивну веб-сторінку моніторингу насосної станції водопостачання (варіант 15);
  • Динамічне оновлення 4-х параметрів із генерацією випадкових даних;
  • Кольорову індикацію статусів (норма/попередження/критично);
  • Ручне оновлення кнопкою та автоматичне оновлення таймером із вибором інтервалу;
  • Графіки зміни параметрів у часі за допомогою Chart.js;
  • Журнал вимірювань зі збереженням у localStorage та експортом у CSV;
  • Додаткові індикатори: стан насосів, частота обертання, температура двигуна;
  • Адаптивний дизайн для мобільних пристроїв.

Які навички отримано:

  • Робота з DOM: динамічне створення елементів, оновлення вмісту, зміна класів;
  • Обробка подій: addEventListener, управління потоком через обробники;
  • Таймери: setInterval / clearInterval для періодичного оновлення;
  • Генерація випадкових даних у заданих діапазонах;
  • Інтеграція зовнішніх бібліотек (Chart.js, Bootstrap);
  • Робота з localStorage для збереження даних між сесіями;
  • Експорт даних у CSV з підтримкою Unicode (BOM).

Складнощі та їх вирішення:

  • Коректне відображення кирилиці у CSV-файлі — вирішено додаванням BOM-маркера (\uFEFF);
  • Перезапуск таймера при зміні інтервалу — вирішено через clearInterval перед новим setInterval;
  • Обмеження кількості точок на графіку — використано shift() для видалення старих даних.

Пропозиції щодо покращення:

  • Додати WebSocket-з'єднання для отримання реальних даних від контролера;
  • Реалізувати темну тему з перемикачем;
  • Додати звукові сповіщення для критичних значень;
